# The Thinking Foundry

**The software that teaches people how to think when AI provides answers but not context.**

This is the product repository for The Thinking Foundry — a voice-first interface for navigating complex decisions alongside AI using a battle-tested thinking methodology.

## What This Is

The Thinking Foundry is not a chatbot. It's a 1-3 hour guided thinking session where:
- You bring a messy problem and unclear thinking
- We explore possibilities without jumping to solutions
- We document the entire thinking process in GitHub
- You leave with clarity, a repeatable process, and a GitHub artifact

## Phases

The product follows The Foundry's 8-phase methodology:

### Phase 0: User Stories (ANCHOR)
**Before anything.** What do you want? What's success? What fails? What are your real constraints?

### Phase 1: MINE
**Intake & Diagnosis.** Deep listening. What's the real problem? Not what you thought was wrong.

### Phase 2: SCOUT
**Explore the Possibility Space.** Everything on the table. Sources. Frameworks. Contradictions.

### Phase 3: ASSAY
**Signal from Noise.** What matters for THIS person? Which frameworks fit? Adapt everything.

### Phase 4: CRUCIBLE
**Test Ideas.** War-game scenarios. What breaks? What's fragile? Validate assumptions.

### Phase 5: AUDITOR
**Quality Check.** Are we missing anything? Any logical gaps? Confidence check.

### Phase 6: PLAN
**Clear Answers.** Here's what you should do. Here's why. Here's the GitHub repo proving how we got here.

### Phase 7: VERIFY
**Documentation.** The entire thinking is captured and shareable.

### Phase 8: RALPH-LOOP
**Learning.** What did we learn? How do we improve the process?

## Tech Stack

- **Voice Interface** — Google Gemini Live API (real-time, interruption-enabled)
- **Thinking Space** — GitHub Issues & Repos (public, transparent, teachable)
- **Backend** — TypeScript, Claude API (for analysis), Google Gemini (for voice)
- **Deployment** — Vercel (frontend), Cloudflare Workers (backend)
